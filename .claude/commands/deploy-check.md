Fuehre einen Pre-Deployment-Check durch:

1. **Build testen** — Build-Befehl ausfuehren und pruefen ob er sauber durchlaeuft
2. **Tote Links** — Interne Links auf fehlende Routen pruefen
3. **Env-Check** — Sicherstellen dass keine Secrets im Code sind
4. **Dockerfile** — Pruefen ob Dockerfile konsistent ist und Base-Image aktuell
5. **Git Status** — Uncommitted changes anzeigen, Branch-Info

Zeige am Ende eine Go/No-Go Empfehlung mit Begruendung.
